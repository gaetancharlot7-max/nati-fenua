💳 Sécurité des Paiements (Stripe)

✅ Stripe gère la sécurité des paiements
Les données de carte bancaire ne transitent JAMAIS par votre serveur.
Protection Description Statut
PCI DSS Niveau 1 Stripe est certifié au plus haut niveau ✓ Actif
3D Secure Vérification bancaire supplémentaire ✓ Actif
Tokenisation Cartes remplacées par des tokens ✓ Actif
Radar (Anti-fraude) Détection IA des transactions suspectes ✓ Actif
Webhooks signés Vérification des événements Stripe ✓ Actif

🔧 Action recommandée pour la production

Passez en mode LIVE sur Stripe :

  1. Allez sur dashboard.stripe.com
  2. Activez votre compte (vérification d'identité)
  3. Copiez les clés LIVE (commencent par sk_live_)
  4. Mettez à jour sur Render → Environment Variables

⚠️ Recommandations IMPORTANTES

Action Priorité Comment faire
Activer Cloudflare HAUTE Protection DDoS + WAF gratuit
Changer JWT_SECRET HAUTE Générer une clé de 64+ caractères
Backup MongoDB HAUTE Activer sur MongoDB Atlas
2FA Admin MOYENNE Ajouter authentification 2 facteurs
Logs de sécurité MOYENNE Configurer alertes sur connexions suspectes
Politique de mots de passe BASSE Forcer 8+ caractères, majuscules, chiffres

🛡️ Configuration Cloudflare (GRATUIT)

Cloudflare offre une protection gratuite contre les attaques :

📋 Étapes de configuration

  1. Créez un compte sur cloudflare.com
  2. Ajoutez votre domaine
  3. Changez les DNS chez votre registrar
  4. Activez les protections :
    • ✅ DDoS Protection (automatique)
    • ✅ WAF (Web Application Firewall)
    • ✅ Bot Fight Mode
    • ✅ Rate Limiting
    • ✅ SSL/TLS Full (Strict)
Protection Description
DDoS Bloque les attaques par déni de service
WAF Bloque les injections SQL, XSS, etc.
Bot Protection Bloque les scrapers et bots malveillants
Rate Limiting Limite les requêtes par IP

🔑 Variables d'environnement sécurisées (Render)

⚠️ Ne JAMAIS mettre ces valeurs dans le code !
Utilisez toujours les variables d'environnement Render.
Variable Usage Exemple sécurisé
JWT_SECRET Signature des tokens Générer avec : openssl rand -hex 64
MONGO_URL Base de données mongodb+srv://user:pass@cluster...
STRIPE_SECRET_KEY Paiements sk_live_...
GOOGLE_CLIENT_SECRET OAuth Depuis Google Cloud Console

🔧 Générer un JWT_SECRET sécurisé

openssl rand -hex 64

Copiez le résultat dans Render → Environment Variables

🚨 En cas d'incident de sécurité

  1. Identifiez le type d'attaque (logs Render)
  2. Bloquez les IPs suspectes (Cloudflare)
  3. Changez tous les mots de passe et secrets
  4. Contactez Stripe si paiements compromis
  5. Informez les utilisateurs si données volées (RGPD)
